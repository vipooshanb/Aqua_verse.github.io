Profile Page

Technical Description

The profile page uses HTML5 elements like main and nav for clear structure. A flexbox layout creates a navigation bar and content area. CSS variables set consistent colors and styles. Cards have a glass effect with a blur filter. The availability table is clean with collapsed borders. Media queries ensure it works on mobile devices.

Accessibility

HTML elements help screen readers navigate. Form labels are linked for clarity. Colors are high-contrast for readability. Navigation supports keyboards. The table has a legend for visual clarity, and images include alt text.

Sitemap Page

Technical Description

The sitemap page uses HTML5 elements like main and section for structure. An SVG element displays a visual map with nodes and lines for navigation links. CSS sets a background image with an overlay for contrast. Styles include variables for colors, and nodes use fill and stroke for visual effects. Hover and focus states enhance interactivity. Media queries scale the SVG for smaller screens.

Accessibility

Semantic HTML aids screen readers. SVG nodes have descriptive text for accessibility. High-contrast colors ensure readability. Links support keyboard navigation with focus states. The page is responsive for various devices.

Content Page

Technical Description

The content page uses semantic HTML5 with Flexbox and Grid layouts to organize initiative cards in a responsive design. Each card includes an image, title, description, and a button, styled with consistent CSS variables and transitions for interactivity. A fixed background video with an overlay enhances visual appeal while keeping text readable. Media queries ensure the layout adapts smoothly across devices.

Accessibility

Accessibility was prioritized using semantic tags, alt text for all images, and strong color contrast. Interactive elements are keyboard-navigable with clear focus indicators. Icons include aria-labels, and the design supports screen readers and zooming without breaking the layout.

Challenges and Lessons Learned

Implementing the Profile and Sitemap pages presented challenges. For the Profile page, aligning the flexbox layout for responsiveness across devices was tricky, especially balancing the navigation bar and content area. Adjusting media queries to scale images and padding solved this, teaching the importance of testing on multiple screen sizes. The Sitemap page’s SVG nodes were initially misaligned, requiring tweaks to coordinates and responsive scaling. This improved understanding of SVG positioning. Accessibility was another challenge; ensuring screen reader compatibility for the SVG and form elements required adding descriptive text and labels. These efforts reinforced the value of iterative testing and user-focused design.

Compliance

The Profile and Sitemap pages comply with Janet regulations by using semantic HTML for structure, ensuring accessibility with high-contrast colors and keyboard navigation. Alt text for images and descriptive SVG labels support screen readers, meeting accessibility standards. Responsive design ensures usability across devices. Valid HTML and CSS were confirmed through validation tools, aligning with regulations for clean, error-free code. Content is clear and appropriate, adhering to publication guidelines.
